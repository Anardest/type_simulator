# Документация: Эмулятор ввода текста с клавиатуры

Скрипт для автоматического ввода больших объёмов текста в программы, которые принимают только реальные нажатия клавиш и не поддерживают вставку через буфер обмена. Эмулирует посимвольный ввод с заданной задержкой.

## Требования

- **Python** версии 3.1 или выше.
- Операционная система: Windows, macOS, Linux.
- Библиотека `pyautogui` (управление клавиатурой и мышью).

---

## Установка и настройка окружения

Рекомендуется использовать виртуальное окружение для изоляции зависимостей.

### Создание виртуального окружения (venv)

Откройте терминал (командную строку) в папке, где будет лежать скрипт, и выполните:

```bash
python -m venv venv
```
### Активация окружения
Затем в том же терминале (командой строке) выполните команду:

```bash
venv/Scripts/activate
```

### Установка зависимостей

Установите зависимость:

```bash
pip install pyautogui
```

### Использование скрипта

1. Запустите скрипт:

Ввод из файла:

```bash
python type_simulator.py -f my_text.txt -d 0.01 -w 3
```

Ввод текста напрямую:

```bash
python type_simulator.py -t "Hello, world!"
```

2. Переключитесь на окно программы-приёмника в течение заданного времени ожидания (по умолчанию 5 секунд).
3. Скрипт начнёт эмулировать нажатия клавиш один за другим.


### Аргументы командной строки

| Аргумент  | Описание |
| ------------- | ------------- |
| -f, --file FILE  | Путь к текстовому файлу, содержимое которого будет введено.  |
| -t, --text TEXT  | Строка текста для ввода (если текст короткий).  |
| -d, --delay SEC  | Задержка между нажатиями клавиш в секундах (по умолчанию 0.02).  |
| -w, --wait SEC  | Время ожидания перед началом ввода, чтобы переключиться на целевое окно (по умолчанию 5).  |
	
Если не указан ни -f, ни -t, скрипт читает текст из стандартного ввода (stdin).

### Примеры запуска

1. Ввод из файла с задержкой 0.01 сек и ожиданием 3 секунды:
```bash
python type_simulator.py -f my_long_text.txt -d 0.01 -w 3
```
2. Ввод короткой строки напрямую:
```bash
python type_simulator.py -t "Hello, world!"
```
3. Чтение текста через конвейер (pipe):
```bash
echo "Текст для ввода" | python type_simulator.py
```
4. Интерактивный ввод (завершить Ctrl+D на Linux/macOS или Ctrl+Z Enter на Windows):
```bash
python type_simulator.py
Введите текст (завершите ввод Ctrl+D / Ctrl+Z):
> Введите ваш большой текст...
```

### Важные замечания

Раскладка клавиатуры должна соответствовать языку вводимого текста (например, русская раскладка для кириллицы). pyautogui отправляет символы согласно текущей активной раскладке.

Для очень больших объёмов текста (миллионы символов) можно увеличить --delay или использовать альтернативные библиотеки (keyboard, pynput), но pyautogui справляется с большинством задач.

На некоторых системах (Linux, macOS) могут потребоваться дополнительные разрешения на управление вводом. На Windows проблем обычно нет.

### Устранение неполадок
| Проблема | Решение |
|----------|---------|
| `ModuleNotFoundError: No module named 'pyautogui'` | Установите библиотеку: `pip install pyautogui` внутри активированного venv. |
| Символы вводятся неправильно (например, вместо `@` вводится `"`) | Проверьте раскладку клавиатуры. Переключите её на нужный язык перед запуском скрипта. |
| На macOS скрипт не вводит текст | Перейдите в *Системные настройки → Безопасность → Конфиденциальность → Управление клавиатурой* и разрешите доступ для терминала или приложения, из которого запущен Python. |
| На Linux символы не вводятся в некоторых приложениях (например, в игре) | Попробуйте запустить скрипт с правами суперпользователя (не рекомендуется) или используйте альтернативную библиотеку `python-xlib` + `pynput`. |
| Задержка слишком большая/маленькая | Настройте `--delay`. Для быстрого ввода – `0.005`, для надёжного – `0.03–0.05`. |
| Текст не вводится целиком, обрывается | Увеличьте `--delay` или разбейте текст на части с помощью пауз между вызовами скрипта. |
